# Production Pack 004: The Monastic MCP & HE v3.0 Bridge

## Overview
This pack explains how to bridge a legacy REST game (Abadía del Crimen) with modern AI Agent architectures using the **Model Control Program (MCP)** and **Harness Engineering v3.0**.

## Key Concepts
1. **The MCP as the "Holy Rules":** Just as the Rule of Saint Benedict governs the monastery, the MCP Server governs the agent's interaction with the game.
2. **From Vibe to Spec:** Moving from "sending UP commands and hoping" to a deterministic 2x2 volume validation.
3. **The Dreamer Loop:** How to use session logs to "dream" about the abbey and update our mental map (The Hub).

## Book Metaphors (HE v3.0)
- **The Abbot (The Sensor):** The entity that verifies compliance. In HE, this is our test suite.
- **The Scriptorium (The Hub):** Where knowledge is recorded. In HE, this is the interlinked Wiki.
- **The Library (The Context):** A vast, labyrinthine source of information that must be explored carefully to avoid "context rot".
