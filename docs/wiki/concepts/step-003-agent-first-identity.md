# Step 003: Agent-First Identity

Transitioning the project constitution from a tool-specific name (`CLAUDE.md`) to a role-specific, agnostic anchor: **`AGENTS.md`**.

## The Why
Relying on a tool-specific file name creates a hierarchy where the agent is a guest in the tool's world. By using `AGENTS.md`, we establish an **Agent-Native** environment where the project's laws (HE v3.0) are supreme, regardless of whether the executor is Gemini, Claude, or a custom script via Antigravity.

## Technical Concept
- **Agent Sovereignty:** The principle that the project harness must be model-agnostic and self-describing.
- **Universal Anchor:** `AGENTS.md` acts as the first point of entry for any autonomous entity, providing a clear mapping of the toolchain (`gemini-cli`, `antigravity`) and standards.

## The Lesson
In the era of Software 3.0, we don't write instructions for a model; we write a **Manual of Operations** for the system's agents. This shift from "Commanding" to "Orchestrating" is the heart of Harness Engineering.
