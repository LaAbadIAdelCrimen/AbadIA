# Step 003: Master Manifesto

Establishing the `CLAUDE.md` file as the central nervous system for multi-agent coordination.

## The Why
In a multi-agent environment (Hermes, Claude Code, Gemini-cli), context drift is the primary enemy. The `CLAUDE.md` acts as a static "Anchor" that persists across sessions and models, ensuring that every agent adheres to the same HE v3.0 standard.

## Technical Concept
- **Context Anchoring:** Using a root-level markdown file to define project identity and behavioral constraints.
- **Polyglot Harness:** Designing instructions that are interpreted equally well by reasoning-first models (Claude) and speed-first models (Gemini).

## The Lesson
The file is named `CLAUDE.md` because it is the industry standard for Claude Code, but its function is that of a **Project Constitution**. It is where we "grabamos en piedra" the Beyoncé Rule and the Karpathy Laws.
